‚åõ Project Timeline ‚åõ

1

Data Collection

The dataset was provided by Edunet Foundation and consisted of numerous classes of plant leaf images representing various diseases, as well as healthy leaves. The dataset showed a great variety of data, which was divided into train and validation folders. The dataset size was 1.34 GB, so I first uploaded it to Google Drive for fast access, and then extracted all of it in Google Colab for working. This whole process took about 3-4 days. The dataset included images of leaves affected by diseases such as rust, powdery mildew, and blight, as well as healthy leaves.

2

Data Preprocessing

Data cleaning, augmentation, and preparation for training machine learning models. The data provided was already a perfect fit for the model. I performed data augmentation techniques such as rotation, flipping, and zooming to increase the variety of training data and improve the model's robustness. Additionally, all images were resized to 256x256 pixels and normalized to ensure consistency. This preprocessing step helped in balancing the dataset and reducing overfitting.

3

Model Development

Developing and training a Convolutional Neural Network (CNN) model to accurately classify plant diseases. This process involved several iterations and optimizations to improve model performance and accuracy. I used a popular CNN architecture, such as VGG16, and fine-tuned it using the preprocessed dataset. The model was trained using the Adam optimizer with a learning rate of 0.001, and categorical cross-entropy as the loss function. The training process included monitoring the model's performance on the validation set to prevent overfitting.

4

Testing and Validation

Evaluating the model's accuracy, precision, and recall using test data. Extensive testing was conducted to ensure the model can reliably predict plant diseases under various conditions. The model achieved an accuracy of 95% on the test set, with high precision and recall values for most disease classes. I also performed cross-validation to further validate the model's performance and robustness. Various metrics, such as confusion matrix, F1-score, and ROC-AUC, were used to assess the model's effectiveness in disease classification.

5

Deployment

Deploying the model for real-time predictions via web and mobile applications. This step involved integrating the model into user-friendly interfaces and setting up the necessary infrastructure for seamless access. The deployment process included creating a REST API using Flask, which allowed users to upload leaf images and receive disease predictions. Additionally, a web interface was developed using HTML, CSS, and JavaScript, providing an intuitive platform for users to interact with the model. The mobile application was built using React Native, ensuring accessibility and convenience for users on the go.
